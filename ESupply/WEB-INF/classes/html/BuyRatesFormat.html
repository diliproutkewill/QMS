Buy Rates - File Format
Important Points
1 The user can uplaod buy rates for Shipment Mode - Air, Sea, Truck
2 The formats differ for different Weight Breaks - FLAT, SLAB, BOTH
3 For each weight break the data is entered under 2 headers
4 First header is used to enter MASTER data for the buy rates
e.g SHIPMENT_MODE* CURRENCY* WEIGHT_BREAK* WEIGHT_CLASS* RATE_TYPE* UOM CONSOLETYPE DENSITY RATIO*
AIR AED FLAT General FLAT KG
One Empty row should be there.
5 Second header is used to store the DETAILS data for the buy rates
e.g ORIGIN DESTINATION CARRIER_ID* SERVICE_LEVEL FREQUENCY TRANSIT TIME EFFECTIVE_FROM VALID_UPTO MIN FLAT FSMIN FSKG SSMIN SSKG NOTES
ACA AKL C1 EC 1&2 12:00 1-Sep-05 30-Sep-05 12 5 6 3 4 7 9 aaaaa
ACA ALY C2 EC 1&2 12:00 1-Sep-05 30-Sep-05 12 5 6 3 4 7 9 aaaaa
ACA AKL C3 EC 1&2 12:00 1-Sep-05 30-Sep-05 12 5 6 3 4 7 9 aaaaa
ACA ALY C4 EC 1&2 12:00 1-Sep-05 30-Sep-05 12 5 6 3 4 7 9 aaaaa
6 The user can enter any number of DETAILS data
7 For Weight break - SLAB, Rate Type - FLAT/SLAB, the upload format is same
8 For Weight break - SLAB, Rate Type - BOTH; the user needs to specify a suffix F or S after the buy rate as per requirement
9 For Air - LIST, the user can specify an Over-Pivot rate for each tradelane
10 If there are no buy rates for a particular weight break, the user needs to specify NA (Not Applicable) else the system will give error if left blank
11 There are no limitation for the number of weight breaks for SLAB
12 For Sea & Truck, the Console Type is mandatory (LCL/FCL, LTL/FTL)
13 For Sea - LIST (FCL) & Truck - LIST (FTL), pivot rates are not applicable
14 FREQUENCY
For Air/Truck, the user cannot specify the days of the week with comma separation. Instead, & (ampersand) separator can be used.
For Sea, the user can specify only the valid values - WEEKLY, FORTNIGHTLY, MONTHLY, EVERY 10 DAYS; any other value will give an error.
The system checks for unique combination based on origin, destination, carrier, service level & frequency
15 TRANSIT TIME
For Air, the transit time must be in HH:MM format
For Sea, the transit time can be specified in days
16 LIST - the system will check for valid list values for the shipment mode from the LIST MASTER
17 Density Ratio: The user must define the valid density ratio in the header level which applies to all the rows of Buy Rates in detail level.
The system checks for the valid values of Density code entered based on the shipment mode in Density Group code master
18 In second header, Surcharges weight breaks that can be used for AIR :FSBASIC,FSMIN,FSKG,SSBASIC,SSMIN,SSKG
Surcharges weight breaks that can be used for SEA :BAFMIN,BAFM3,CAFMIN,CAF%,CSF,PSSMIN,PSSM3
Surcharges weight breaks that can be used for TRUCK :SURCHARGE
19 We can use NA for notes if it is not applicable or it can be left blank.
